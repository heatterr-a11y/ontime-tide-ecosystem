## 👁️ The Vision
Current digital systems wait for a user to click. **OnTime-TIDE™** senses the intent *before* the action. By identifying the **1.8s Absence of Action**, we synchronize digital outcomes in a **120ms window**, creating a frictionless interface between human thought and machine execution.

## 🛠️ The Three-Pillar Architecture

### 1. DTIF (Decentralized Temporal Intent Framework) - "The Brain"
The sensing layer that monitors user dwell-time and receptivity.
* **Core Logic:** Identifies the 1.8s threshold where "browsing" turns into "deciding."
* **File:** `/dtif-brain/IntentScarcityAlpha.java`

### 2. TISE (Temporal Intent Synchronization Engine) - "The Engine"
The high-speed execution core that bridges the gap between intent and settlement.
* **Core Logic:** A 120ms micro-decision window that processes the Receptivity Vector (V).
* **File:** `/tise-engine/TiseCoreEngine.java`

### 3. TIDE (The Protocol) - "The Economy"
The $ONTM settlement layer that incentivizes high-velocity participation.
* **Mechanism:** 70% Token Burn (Scarcity) / 30% User Rebate (Receptivity).
* **File:** `/tide-protocol/OnTimeTideProtocol.sol`
